## Architecture

```
User query (English or Chinese)
        │
        ▼
parse_query_with_llm()  ──── fallback ──→  parse_query_rules()
  [tools/parser_tools.py]                  (regex, no LLM)
        │
        ├──→ ask_clarifying_questions()      [Component 1]
        │      If query is vague, ask up to 3 follow-up Qs.
        │      User answers → apply_clarification_answers() → re-parse.
        │
        ▼
  intent dict
  {cancer_type, sample_types, platform, year_start, year_end, ...}
        │
        ├──→ build_pubmed_query_with_controls()  [Component 3]
        │      Returns {main, cancer_only, control_only} query strings.
        │
        ▼
  tools/ncbi_search.py
        │
        ├── esearch_pmids()   → PMID list (per variant query)
        ├── efetch_abstracts() → [{pmid, title, abstract}]
        ├── stage1_filter()   → keyword scan (methylation + cfDNA/plasma,
        │                        exclude cell lines / mouse models)
        └── stage2_extract()  → extract_paper_structured() [Component 5]
                                 → structured records
        │
        ▼
  Sorted by confidence_level (high → medium → low)
        │
        ├──→ evaluate_geo_dataset()  [Component 4]
        │      For each GSE accession found, judge keep/exclude/manual_review
        │
        ▼
  Registry (registry/methyagent.db)   ← not yet wired
        │
        ▼
  GEO download  (tools/geo_tools.py)  ← not yet wired
```

## Component Summary

### Component 1 — `ask_clarifying_questions(query, llm)`
`tools/query_clarifier.py:94`

Sends the raw user query to the LLM with a system prompt that identifies ambiguous dimensions (cancer type, sample type, controls, platform, year). Returns a `ClarificationResult` dataclass with up to 3 `ClarifyingQuestion` objects.

A query is **specific enough** (no questions asked) when all three core dimensions are present: cancer type + sample type + controls. Platform and year are optional.

Chinese terms are fully recognised — e.g. `结直肠癌` = colorectal, `血浆cfDNA` = cfDNA/plasma, `需要健康对照` = controls required.

- **Input:** raw query string
- **Output:** `ClarificationResult(is_specific_enough, questions, missing_dimensions, clarified_intent_hint)`
- **Status:** PASS (3/3 test cases)

### Component 2 — `build_ncbi_safe_pubmed_query(intent)`
`tools/query_clarifier.py:260`

Assembles a PubMed boolean query from parsed intent. Enforces NCBI limits: max 5 OR-terms per AND-group, max 4 AND-groups, max 400 chars.

- **Input:** intent dict
- **Output:** single PubMed query string
- **Status:** tested offline via smoke test

### Component 3 — `build_pubmed_query_with_controls(intent)`
`tools/query_clarifier.py:427`

Wraps Component 2 to produce three variants:
- `main` — base + control clause
- `cancer_only` — base + detection/screening terms
- `control_only` — base + healthy-donor / normal-plasma terms

- **Input:** intent dict
- **Output:** `{"main": str, "cancer_only": str, "control_only": str}`

### Component 4 — `evaluate_geo_dataset(dataset_info, cancer_type, sample_types, llm)`
`tools/query_clarifier.py:544`

LLM judge for GEO datasets. Returns structured dict with `recommended_action: keep | exclude | manual_review | article_only`.

- **Status:** PASS (3/3 test cases)

### Component 5 — `extract_paper_structured(abstract, llm, pmid, title)`
`tools/query_clarifier.py:643`

LLM extractor for PubMed abstracts. Returns:

| Field | Description |
|-------|-------------|
| `cancer_type` | CRC / lung / breast / … |
| `sample_type` | tissue / plasma_cfdna / serum_cfdna / wbc / … |
| `has_normal_control` | bool |
| `sample_size_case/control` | int |
| `dataset_ids` | GSE…, TCGA-… |
| `markers_or_panel` | [{id, gene, type}] |
| `performance_metrics` | {auc_training, auc_validation, auc_external, sensitivity_at_specificity} |
| `data_availability` | public_download / controlled_access / … |
| `confidence_level` | high / medium / low |

- **Status:** PASS (3/3 test cases)

## `tools/ncbi_search.py`

### Functions

| Function | Description |
|----------|-------------|
| `esearch_pmids(query, max_results)` | NCBI esearch → PMID list |
| `efetch_abstracts(pmids)` | NCBI efetch XML → `[{pmid, title, abstract}]` |
| `fetch_pubmed_records(query, max_results)` | esearch + efetch combined |
| `stage1_filter(records)` | Rule-based: keep if methylation + cfDNA/plasma; drop cell lines |
| `stage2_extract(records, llm)` | Calls `extract_paper_structured()` per record, capped at 10 |
| `search_and_extract(intent, llm, top_n)` | Full pipeline — 3 query variants, dedup, Stage 1 → 2 → sort |

### Stage 1 keyword rules

**Keep requires:**
- Methylation keyword: `methylation | methylated | methylome | cpg | bisulfite | epic | 450k | wgbs | rrbs | mcta`
- Liquid biopsy keyword: `cfdna | cell-free dna | ctdna | plasma | liquid biopsy | serum | circulating dna`

**Exclude if any:**
- `cell line | in vitro | xenograft | pdx | organoid | mouse model | murine | dnmti | 5-azacytidine | decitabine | drug treatment | radiation treatment`

### Rate limits & timing
- Without `NCBI_API_KEY`: 3 req/s (340ms sleep enforced)
- With `NCBI_API_KEY`: 10 req/s
- Stage 2 LLM extraction: ~3–8s per paper, capped at 10 records → expect 30–80s total

